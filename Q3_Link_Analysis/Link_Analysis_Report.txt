3. Effect of Deleting the Original File
Command: rm original.txt, then cat hardlink.txt, cat symlink.txt
Output: [paste result_after_delete.txt content]
Observation: hardlink.txt still displays content correctly since it references 
the same inode/data. symlink.txt fails with "No such file or directory" since it 
only referenced the now-deleted filename.

4. Metadata Collected
stat output showed file size, inode number, link count, permissions, and 
timestamps for all three files.

5. Conclusion
Hard links are indistinguishable from the original at the filesystem level since 
they share an inode. Symbolic links are fragile references that break if the 
target is removed, but are more flexible since they can point across filesystems 
and to directories.
